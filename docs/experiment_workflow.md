# 实验记录工作流

每次实验前、中、后应遵循以下流程，确保实验结果被完整记录。

## 流程

### 1. 实验前：填写计划

在 `docs/experiments/` 下创建新文件（命名规范：`YYYYMMDD_{方法}_{关键词}.md`），复制 `docs/experiment_template.md` 的内容，填写：

- **实验名称**、**日期**、**方法**
- **第 1 节「实验目的」**：为什么做这个实验，预期获得什么结论
- **第 2 节「实验 Setting」**：数据、算法改动、超参

### 2. 实验执行

跑训练或评测。训练配置和评测命令是实验 Setting 的重要组成部分，记录关键参数。

### 3. 实验后：立即记录

- **第 3 节「实验结果」**：填入定量结果（success rate 等）和定性观察（好坏 case）
- **第 4 节「分析」**：结果是否与预期一致，不一致的原因分析
- **第 5 节「Next Steps」**：基于当前结果，主动规划下一步实验

### 4. 检查清单

- [ ] 实验报告文件已创建在 `docs/experiments/` 下
- [ ] 六项（目的/Setting/结果/分析/Next Steps）完整填写
- [ ] 量化结果附有具体数字，非模糊描述
- [ ] Next Steps 有优先级排序
